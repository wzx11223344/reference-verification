# 参考文献真伪核验技能

## 概述

本技能提供系统化的学术参考文献真伪核验工作流，覆盖文献信息提取、多平台交叉检索、逐条真实性核验、异常问题归因、核验报告生成五个核心阶段。支持中英文混合文献列表，适配期刊论文、会议论文、学位论文、书籍、专利、网络资源等多种文献类型。

---

## 工作流程

### 阶段一：文献条目识别与结构化提取

1. **扫描目标材料**，定位参考文献列表（通常位于文末、脚注或参考文献章节）
2. **逐条提取**以下字段，整理为结构化表格：

| 字段 | 说明 | 示例 |
|------|------|------|
| 编号 | 原文标注序号 | [1] / ①  |
| 文献题名 | 完整标题 | "Deep Learning for NLP" |
| 全部作者 | 姓名及顺序 | Zhang, Y.; Li, X. |
| 发表载体 | 期刊名/会议名/书名 | IEEE TPAMI |
| 年份 | 发表年份 | 2023 |
| 卷/期/页码 | Vol., No., pp. | Vol.45, No.3, pp.1024-1038 |
| DOI/ISBN | 唯一标识符（如有） | 10.1109/TPAMI.2023.XXXXXX |

3. **特殊情况处理**：
   - 格式不规范条目：尽量还原字段，标注「格式异常」
   - 仅有网址的条目：记录URL及访问描述
   - 引用书目：额外记录出版社、城市信息

---

### 阶段二：多平台学术检索

针对每条文献，按以下优先级顺序检索，命中即停止：

#### 优先检索平台（通用）

| 平台 | 适用范围 | 搜索方式 |
|------|---------|---------|
| Google Scholar | 全领域覆盖最广 | `site:scholar.google.com` 搜索题名+作者 |
| Semantic Scholar | 全领域，AI辅助 | `https://api.semanticscholar.org/graph/v1/paper/search?query=` |
| CrossRef | DOI权威查询 | `https://api.crossref.org/works/{DOI}` |
| PubMed | 生命科学/医学 | `https://pubmed.ncbi.nlm.nih.gov/?term=` |

#### 专业数据库（按学科）

| 平台 | 适用学科 | 检索URL前缀 |
|------|---------|------------|
| IEEE Xplore | 电气/电子/计算机 | `https://ieeexplore.ieee.org/search/searchresult.jsp?queryText=` |
| ACM Digital Library | 计算机科学 | `https://dl.acm.org/search/?query=` |
| arXiv | 预印本（CS/Math/Physics） | `https://arxiv.org/search/?query=` |
| CNKI 知网 | 中文文献 | `https://www.cnki.net/` |
| 万方数据 | 中文文献 | `https://www.wanfangdata.com.cn/` |
| Web of Science | 多学科高质量期刊 | SCI/SSCI收录核验 |
| Springer/Nature | 自然科学 | `https://link.springer.com/search?query=` |
| Elsevier/ScienceDirect | 多学科 | `https://www.sciencedirect.com/search?qs=` |

#### 检索策略

```
主检索词：完整题名（去除特殊符号后加引号精确匹配）
辅助检索：第一作者姓名 + 年份
备用检索：关键词组合（题名前3-5个关键词）
DOI直接验证：若有DOI，优先通过 https://doi.org/{DOI} 直接跳转
```

### 阶段三：逐条核验标准

对每条文献，依次核验以下五项：

#### ✅ V1 — 文献真实存在性
- 该文献是否在任意权威数据库中有收录记录
- 判定依据：找到官方发布页面或DOI链接 = 存在；全网无任何记录 = 疑似虚构

#### ✅ V2 — 题名准确性
- 检索到的实际题名与原文引用题名是否一致
- 允许细微差异：大小写、标点、冠词省略
- 不允许：关键词替换、意思改变、题名与实际不符

#### ✅ V3 — 作者信息准确性
- 作者姓名、顺序是否与实际发表一致
- 重点核查：第一作者、通讯作者是否被替换

#### ✅ V4 — 出处信息准确性
- 期刊/会议名称是否正确
- 年份是否正确
- 卷、期、页码是否与实际一致
- DOI（如有）是否能正常解析

#### ✅ V5 — 载体合规性
- 期刊是否为真实存在的正规刊物（非掠夺性期刊）
- 会议是否在官方列表中（非伪造会议）
- 可通过 Beall's List、COPE 成员目录 交叉验证

### 阶段四：异常问题分类

| 问题代码 | 问题类型 | 描述 |
|---------|---------|------|
| `E-FAKE` | 完全虚构 | 全网无任何记录，高度疑似AI/人工编造 |
| `E-TITLE` | 题名错误 | 文献存在但题名有实质性差异 |
| `E-AUTHOR` | 作者错误 | 作者姓名/顺序与实际不符 |
| `E-JOURNAL` | 出处错误 | 期刊/会议名称、卷期页码有误 |
| `E-YEAR` | 年份错误 | 发表年份与实际不符 |
| `E-DOI` | DOI失效 | DOI无法解析或指向不同文献 |
| `E-RETRACT` | 已撤稿 | 文献存在但已被期刊撤回 |
| `E-PREDATORY` | 掠夺性出版 | 刊物涉嫌学术掠夺，质量存疑 |
| `W-UNVERIFIED` | 无法核实 | 检索无结果但不能确认虚假（可能为新文献、内部报告等） |
| `W-FORMAT` | 格式问题 | 引用格式不规范，不影响真实性 |

### 阶段五：核验报告生成

输出完整核验报告，结构如下：

```markdown
# 参考文献真伪核验报告

## 核验概况
- **文件名称**：[被核验文件]
- **核验时间**：[日期]
- **文献总数**：N 条
- **核验完成**：N 条（可能有 N 条无法核实）

## 统计摘要
| 类别 | 数量 | 占比 |
|------|------|------|
| ✅ 真实有效 | N | XX% |
| ❌ 虚假/编造 | N | XX% |
| ⚠️ 信息有误 | N | XX% |
| 🔍 无法核实 | N | XX% |

## 核验明细表
（逐条列出，包含：编号、题名、核验结果、问题代码、问题说明、参考链接）

## 高风险异常条目详析
（对 E-FAKE、E-AUTHOR 等严重问题条目，单独展开说明）

## 总体评估与建议
（综合评价文献列表整体质量，给出修改建议）
```

## 执行指南

### 输入材料处理

- **PDF/Word文件**：直接解析文末参考文献列表
- **截图/图片**：先OCR识别文字，再进行结构化提取
- **纯文本粘贴**：直接解析，注意区分不同引用格式（APA/MLA/Chicago/GB/T 7714）
- **已有编号列表**：保持原编号，逐条核验

### 检索执行要点

1. 对每条文献，先用 **WebSearch** 工具搜索：`"[完整题名]" [第一作者姓] [年份]`
2. 若搜索结果命中，访问官方页面（**WebFetch**）确认详细信息
3. 有 DOI 的文献，直接 `WebFetch https://doi.org/{DOI}` 验证
4. 中文文献优先使用知网/万方搜索
5. 搜索无结果时，尝试缩短题名（取核心词），或换用作者+年份组合

### 报告精度控制

- 有直接证据（官方页面链接）= 标记为 ✅ 真实
- 搜索命中但页面信息与引用有差异 = 标记 ⚠️ 信息有误
- 多平台搜索均无结果 = 标记 ❌ 疑似虚假（附注「建议人工复核」）
- 不确定情况 = 标记 🔍 无法核实，说明原因

### 批量处理策略

- 文献 ≤10 条：逐条完整核验
- 文献 11-30 条：逐条核验，搜索策略可适当精简
- 文献 >30 条：重点核验可疑条目（格式异常、孤僻来源、近年AI高发场景），其余抽样核验，在报告中说明抽样比例
